# Documentation Index

This folder is organized around the current alpha workflow. Top-level docs should stay useful for active development, release packaging, alpha testing, or handoff. Older phase-specific notes are kept under `archive/` so history is still available without crowding the main docs list.

## Start Here

- `PROJECT_STATE.md` - current phase, current capability, validation status, and next work.
- `NEXT_SESSION_HANDOFF.md` - where the next Codex/development session should start.
- `README.md` at the repo root - user-facing project overview and quick commands.

## User And Alpha Testing

- `ALPHA_QUICK_START.md` - first-run instructions for alpha testers.
- `ALPHA_TEST_PLAN.md` - scenarios and artifacts to collect for alpha validation.
- `KNOWN_ISSUES.md` - release-facing limitations and troubleshooting.
- `FEEDBACK_TEMPLATE.md` - what testers should attach when reporting issues.
- `VIEWER_QUICK_START.md` - packaged Viewer instructions used by publish scripts.

## Developer Reference

- `JSON_SCHEMA.md` - current export JSON format.
- `ROUTE_GEOMETRY_NOTES.md` - consolidated pathPoints, CurveElement, track discovery, and route geometry validation notes.
- `DEV_NOTES.md` - chronological implementation notes and local commands.
- `DECISION_LOG.md` - important project decisions and reasons.
- `DEVELOPMENT_PLAN.md` - original long-form plan; useful as historical direction, not a strict current checklist.
- `..\scripts\README.md` - script index grouped by release, validation, diagnostics, and local CS2 mod workflows.

## Release

- `CHANGELOG.md` - release changes.
- `RELEASE_CHECKLIST.md` - future release checklist.
- `ROADMAP.md` - concise roadmap.

## Archived Phase Notes

The `archive/` folder keeps documents that were useful for a specific phase but are no longer the best entry point:

- `archive/PHASE_3C_MEMO.md`
- `archive/VIEWER_MANUAL_TEST.md`
- `archive/PATH_GEOMETRY_VALIDATION.md`
- `archive/PATH_GEOMETRY_VALIDATION_RESULTS.md`
- `archive/METRO_TRACK_GEOMETRY_DISCOVERY.md`

Use the archived files for historical detail. Use the top-level consolidated docs for current work.
